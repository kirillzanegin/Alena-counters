# 🚨 СРОЧНО: Ошибка входа после включения RLS

## Что случилось?

После включения RLS система не пускает вас - ошибка авторизации.

**Причина:** Политики RLS не могут найти вашу запись из-за проблемы с `auth_user_id`.

---

## ⚡ ШАГ 1: СРОЧНО отключить RLS (прямо сейчас!)

### Откройте Supabase SQL Editor и выполните:

```sql
ALTER TABLE employees DISABLE ROW LEVEL SECURITY;
```

Или используйте файл: **`SQL_DISABLE_RLS_NOW.sql`**

✅ **После этого вы сможете войти снова!**

---

## 🔍 ШАГ 2: Диагностика - Найти проблему

### Выполните файл: **`SQL_DIAGNOSE_RLS.sql`**

Скопируйте весь код и выполните в SQL Editor.

### Что искать:

1. **Проверка 1**: Есть ли `auth_user_id` в таблице `employees`?
   - Если NULL → это проблема!

2. **Проверка 5**: Совпадают ли `auth_user_id` из `employees` с `id` из `auth.users`?
   - Если НЕ совпадают → это проблема!

### Пришлите мне скриншот результата этого запроса:

```sql
SELECT 
  e.id,
  e.email,
  e.auth_user_id as employee_auth_user_id,
  u.id as auth_users_id,
  CASE 
    WHEN e.auth_user_id = u.id THEN '✅ Совпадает'
    WHEN e.auth_user_id IS NULL THEN '❌ NULL в employees'
    ELSE '❌ НЕ совпадает!'
  END as status
FROM employees e
LEFT JOIN auth.users u ON e.email = u.email
ORDER BY e.created_at DESC;
```

## 🎯 Возможные причины проблемы

### Причина 1: `auth_user_id` пустой (NULL)

Когда создавали пользователей через `admin-create-user`, поле `auth_user_id` могло не заполниться.

**Как исправить:**
```sql
-- 1. Найти правильный UUID из auth.users
SELECT id, email FROM auth.users WHERE email = 'ваш_email@example.com';

-- 2. Обновить employees
UPDATE employees 
SET auth_user_id = 'правильный_uuid_из_auth_users'
WHERE email = 'ваш_email@example.com';
```

### Причина 2: `auth_user_id` не совпадает

Если в `employees.auth_user_id` один UUID, а в `auth.users.id` другой.

**Как исправить:** Синхронизировать вручную (см. выше).

### Причина 3: Пользователь не аутентифицирован правильно

Политики проверяют `auth.uid()`, но он возвращает NULL или неправильное значение.

**Как проверить:**
```sql
SELECT auth.uid();
```

Если возвращает NULL → проблема с аутентификацией Supabase Auth.
