# ✅ Включение RLS обратно после успешного теста

## Зачем это нужно?

**RLS (Row Level Security)** - это защита на уровне базы данных. Без неё любой пользователь может читать и изменять данные других пользователей. Это **критично для безопасности**!

---

### Шаг 1: Откройте Supabase SQL Editor

1. Зайдите на https://supabase.com
2. Выберите ваш проект
3. Слева: **SQL Editor**
4. Нажмите **"New query"**

### Шаг 2: Скопируйте и выполните SQL

Откройте файл: **`SQL_ENABLE_RLS_BACK.sql`**

Нажмите **"Run"** (▶️)

### Шаг 3: Проверьте результат

```

----------------------------------------|--------|---------------
Users can view own record               | SELECT | authenticated
Users can update own link_token         | UPDATE | authenticated

```

И проверку статуса RLS:

```
-----------|------------

```

## 🛡️ Что делают политики?

### 1. **"Users can view own record"**
   - Пользователь видит только свою запись в таблице `employees`
   - Не может видеть других пользователей

### 2. **"Users can update own link_token"**
   - Пользователь может обновлять свои `link_token` и `link_expires_at`
   - Нужно для кнопки "Привязать Telegram"
   - Не может менять свой `tg_id` напрямую

### 3. **"Owners can view all active employees"**
   - Владельцы видят всех активных сотрудников
   - Нужно для раздела "Управление пользователями"
   - Могут видеть email, имена, роли

### 4. **Edge Functions (service_role) обходят RLS**
   - `telegram-webhook` - обновляет `tg_id` при привязке
   - `admin-create-user` - создаёт новых пользователей
   - `admin-update-user` - обновляет данные пользователей

## ❌ Если что-то сломалось

### Проблема: "Не могу обновить link_token"

**Причина:** Политика `UPDATE` не создана или неправильная.

**Решение:** Выполните снова:
```sql
CREATE POLICY IF NOT EXISTS "Users can update own link_token"
ON employees
FOR UPDATE
TO authenticated
USING (auth.uid() = auth_user_id)
WITH CHECK (auth.uid() = auth_user_id);
```

### Проблема: "Owner не видит список пользователей"

**Причина:** Политика для владельцев не работает.

**Решение:** Проверьте, что у вас `role = 'owner'` в таблице `employees`.

### Проблема: "Бот не может привязать аккаунт"

**Причина:** Edge Function использует `service_role`, который **должен** обходить RLS.

**Решение:** Проверьте в Supabase Dashboard:
- Edge Functions → telegram-webhook → Settings
- Убедитесь, что переменная `SUPABASE_SERVICE_ROLE_KEY` установлена
