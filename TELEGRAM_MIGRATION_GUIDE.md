# Миграция на упрощённую архитектуру Telegram-привязки

## Что изменилось

Убрали отдельную таблицу `telegram_link_tokens` и сложный flow с кодами. Теперь:

1. **Автоматическая привязка из Telegram WebApp**: Когда пользователь открывает приложение из Telegram и входит в систему, его Telegram ID автоматически определяется через `Telegram.WebApp.initDataUnsafe.user.id` и записывается в `employees.tg_id`.

2. **Привязка через токен**: Если пользователь открыл приложение НЕ из Telegram (обычный браузер), он может в разделе "Telegram":
   - Нажать кнопку "Привязать Telegram"
   - Получить ссылку с токеном вида `https://t.me/your_bot?start=TOKEN`
   - Перейти по ссылке в Telegram
   - Бот автоматически привязывает его аккаунт

3. **Никаких кодов и экранов ввода**: Удалён компонент `TelegramCodePrompt`, который требовал от пользователя вводить 6-значный код.

## Что нужно сделать для активации

### 1. Запустить SQL миграцию

Выполните скрипт `SQL_TELEGRAM_SIMPLIFY.sql` в Supabase SQL Editor:

```bash
# Откройте Supabase Dashboard → SQL Editor → New query
# Вставьте содержимое SQL_TELEGRAM_SIMPLIFY.sql и выполните
```

Это добавит в таблицу `employees` колонки:
- `link_token` - токен для привязки (генерируется в приложении)
- `link_expires_at` - срок действия токена

### 2. Деплой обновлённой Edge Function

```bash
cd supabase/functions
supabase functions deploy telegram-webhook
```

### 3. Обновить статические файлы (GitHub Pages или хостинг)

Загрузите обновлённые файлы:
- `index.html` (добавлен Telegram WebApp SDK)
- `app.js` (автоопределение Telegram ID, убран TelegramCodePrompt)

### 4. (Опционально) Удалить старую таблицу telegram_link_tokens

После проверки работы новой системы можно удалить старую таблицу:

```sql
DROP TABLE IF EXISTS telegram_link_tokens CASCADE;
```

⚠️ **Важно**: Сделайте это только после того, как убедитесь, что новая система работает корректно!

## Как это работает сейчас

### Сценарий 1: Пользователь открывает мини-приложение из Telegram

1. Telegram автоматически предоставляет `window.Telegram.WebApp.initDataUnsafe.user.id`
2. Пользователь вводит email и пароль для входа
3. При успешном входе функция `login()` автоматически:
   - Обнаруживает Telegram ID
   - Записывает его в `employees.tg_id` для этого пользователя
4. Готово! Привязка произошла без участия пользователя

### Сценарий 2: Пользователь открывает приложение в браузере

1. Пользователь входит в систему (email + пароль)
2. Переходит в раздел "Telegram"
3. Видит кнопку "Привязать Telegram"
4. Нажимает кнопку → генерируется токен, сохраняется в `employees.link_token`
5. Пользователь видит ссылку `https://t.me/your_bot?start=TOKEN`
6. Переходит по ссылке → бот получает `/start TOKEN`
7. Бот через Edge Function находит сотрудника по токену и записывает `tg_id`
8. Приложение автоматически обнаруживает привязку (polling каждые 3 секунды)
9. Готово!

## Тестирование

1. **Тест автопривязки из Telegram**:
   - Откройте мини-приложение из бота в Telegram
   - Войдите с email/паролем
   - Перейдите в раздел "Telegram"
   - Должно быть: "✅ Telegram привязан"

2. **Тест привязки через токен**:
   - Откройте приложение в обычном браузере (НЕ из Telegram)
   - Войдите
   - Раздел "Telegram" → "Привязать Telegram"
   - Скопируйте ссылку, откройте в Telegram
   - Бот должен ответить: "✅ Аккаунт успешно привязан!"
   - В приложении через несколько секунд статус изменится на "привязан"

3. **Тест отвязки**:
   - Нажмите "Отвязать Telegram"
   - Статус должен измениться на "не привязан"
   - `employees.tg_id` должен стать `NULL`
