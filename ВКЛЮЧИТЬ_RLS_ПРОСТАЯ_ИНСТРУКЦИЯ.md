# ✅ Включение RLS - Простая инструкция

## 🎯 Цель

Включить RLS (Row Level Security) для защиты данных, но так, чтобы:
- ✅ Пользователи могли входить в систему
- ✅ Владельцы видели всех сотрудников
- ✅ Привязка Telegram работала
- ✅ Edge Functions работали

---

## 📋 ШАГ 1: Выполнить SQL

### Откройте Supabase SQL Editor

1. Зайдите на https://supabase.com
2. Выберите ваш проект "Alena"
3. Слева: **SQL Editor**
4. Нажмите **"New query"**

### Скопируйте код

Откройте файл: **`SQL_ENABLE_RLS_SIMPLE.sql`**

Скопируйте **весь код** (Ctrl+A, Ctrl+C) и вставьте в SQL Editor.

### Выполните

Нажмите **"Run"** (▶️) или **Ctrl+Enter**

### Проверьте результат

Вы должны увидеть в конце:

**Таблица 1: RLS включен**
```
tablename  | RLS_enabled
-----------|------------
employees  | true
```

**Таблица 2: Созданы 3 политики**
```
Policy Name                    | Command | Roles
-------------------------------|---------|---------------
policy_employees_select_own    | SELECT  | authenticated
policy_employees_select_owner  | SELECT  | authenticated
policy_employees_update_own    | UPDATE  | authenticated
```

✅ Если видите это - всё отлично!

## 🔍 Что делают политики?

### Политика 1: `policy_employees_select_own`
**Для:** Все аутентифицированные пользователи  
**Что:** Может читать свою запись  
**Зачем:** Для входа в систему и отображения своих данных

### Политика 2: `policy_employees_select_owner`
**Для:** Владельцы (role = 'owner')  
**Что:** Может читать все записи активных сотрудников  
**Зачем:** Для управления пользователями

### Политика 3: `policy_employees_update_own`
**Для:** Все аутентифицированные пользователи  
**Что:** Может обновлять свою запись  
**Зачем:** Для привязки Telegram (обновление `link_token`)

## ⚠️ Что делать, если НЕ работает?

### Вариант А: Временно отключить RLS

```sql
ALTER TABLE employees DISABLE ROW LEVEL SECURITY;
```

После этого вы сможете войти. Напишите мне, и я помогу исправить политики.

### Вариант Б: Проверить текущие политики

```sql
SELECT 
  policyname,
  cmd,
  qual
FROM pg_policies
WHERE tablename = 'employees';
```

Скопируйте результат и отправьте мне.

### Вариант В: Проверить свой auth_user_id

```sql
SELECT 
  id,
  email,
  auth_user_id,
  role,
  is_active
FROM employees
WHERE email = 'ваш_email@example.com';
```

Проверьте, что `auth_user_id` не NULL.
