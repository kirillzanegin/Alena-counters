## 📋 ШАГ 3: Проверить, создаются ли колонки в базе

1. **Откройте** Supabase Dashboard → **SQL Editor**
2. **Скопируйте и выполните** этот SQL:

```sql
SELECT column_name, data_type
FROM information_schema.columns
WHERE table_name = 'employees'
  AND column_name IN ('link_token', 'link_expires_at');
```

### Что должно быть:

```
link_token       | text
link_expires_at  | timestamp with time zone
```

### ❌ Если колонок НЕТ:

Выполните снова:

```sql
ALTER TABLE employees
ADD COLUMN IF NOT EXISTS link_token TEXT,
ADD COLUMN IF NOT EXISTS link_expires_at TIMESTAMP WITH TIME ZONE;
```

## 📋 ШАГ 4: Проверить RLS (Row Level Security)

### Вариант А: Временно отключить (для теста)

В SQL Editor выполните:

```sql
ALTER TABLE employees DISABLE ROW LEVEL SECURITY;
```

⚠️ Это **только для теста**! Потом включим обратно.

### Вариант Б: Добавить правильную политику

```sql
CREATE POLICY IF NOT EXISTS "Users can update own link_token"
ON employees
FOR UPDATE
TO authenticated
USING (auth.uid() = auth_user_id)
WITH CHECK (auth.uid() = auth_user_id);
```
